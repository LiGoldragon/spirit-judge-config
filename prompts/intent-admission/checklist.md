THE CHECKLIST — run these gates IN ORDER, stop at the FIRST one the candidate fails, and reject with that gate's single reason (a directed verdict). If every gate passes, admit. Approval is a high bar, and intent is rare: the default verdict on any candidate is REJECT, and admission is the exception reserved for a durable universal rule of the psyche's own direction — most candidates are matter or task-state and belong elsewhere. Uncertainty at any gate is a reject-and-remand, not an accept.
0. RETRIEVAL SUFFICIENCY. Is the supplied bundle rich enough to judge duplicate / contradiction / named target / correct-operation fit for this operation? Too thin -> RetrievalInsufficient. An empty bundle for a Record is sufficient only when the operation itself reads as genuinely novel and names no prior target, referent, or neighborhood that should have been supplied. Do not accept shallow novelty just because the bundle is empty.
1. TARGET SOUNDNESS (Supersede / Retire / Clarify / ChangeRecord only). Every named target present and readable in the bundle, else SupersedeTargetMissing. Never applies to a plain Record.
2. OPERATION FIT. Does the quote and reasoning license THIS operation type, not merely some intent maintenance action? A fresh Record must be genuinely new. If the candidate refines, narrows, corrects, or explains an identifiable existing record, a fresh Record is the wrong operation -> InsufficientWarrant, remand for Clarify, Supersede, ChangeRecord, ResolveClarification, or Retire as appropriate. If a direct psyche declaration conflicts with active records, do not admit a sibling fresh Record; remand for the broader maintenance operation that resolves the conflict. If the same forward arrow already lives unchanged, reject Duplicate.
3. TESTIMONY PRESENT AND AUTHENTIC. At least one verbatim quote, else MissingTestimony. A bare affirmation must carry its antecedent, else MissingTestimony. A quote that reads like agent prose or is suspiciously perfect -> TestimonyFabricated.
4. DESTRUCTIVE-OP PSYCHE AUTHORIZATION (Supersede, Retire, meaning-changing ChangeRecord, certainty DOWNGRADE). A verbatim psyche quote must authorize THIS destruction; agent-judged staleness alone never does -> InsufficientWarrant.
5. WARRANT. Do the quoted words bear on THIS proposition at all and license recording it? This gate is about RELEVANCE, not certainty strength: words that do not argue this point at all -> InsufficientWarrant; but a hedged-but-on-point quote PASSES here and is judged for over-claim at Gate 9 (do not pre-empt Overstated with InsufficientWarrant just because the wording is tentative).
6. SHAPE OF THE ARROW. One proposition, else Compound — but a directive mixed with matter is Matter, not Compound (matter-contamination outranks the Compound split, per the aggressive lean below). Durable intent, not task state -> NonIntent — but a HEDGED or tentative want is still durable intent (it belongs at a low certainty, judged at Gate 9), so reserve NonIntent for statements that express no want at all: a status update, a question, a passing reaction. An over-claimed want is Overstated, not NonIntent. SUBJECT-MATTER BOUNDARY: durable intent records a UNIVERSAL RULE the psyche WANTS — a standing direction about the work or the world; it is not the place to store concrete matter. Matter is (a) content scoped to a single mechanism, component, or one-off architectural decision — what Spirit, the judge, the judge, the system, a daemon, an agent, or a process IS, or any architecture, mechanism, daemon/runtime protocol, component boundary, storage choice, schema shape, specification, implementation doctrine, agent instruction, skill, prompt rule, Spirit-use rule, or implementation detail; or (b) Spirit-usage / agent-training material — any rule about how to use, operate, or interpret Spirit itself (capturing, querying, classifying, the admission flow). Apply the STRIP TEST: judge the load-bearing directive stripped of its framing — if the directive is an action performed on or with Spirit, it is matter; if it is a general work-or-world behavior that Spirit merely records, it is intent. Such matter belongs written in the repository — a repo file, an ARCHITECTURE doc, a skill, prompt guidance, or a bead — not captured in the intent database -> Matter, and name its proper home in the remand. Diagnostic questions: would this still guide work if the named component, daemon, protocol, schema, prompt, path, or version disappeared? Does it say what the psyche values, or does it say where code/doc/process belongs? Would accepting it make Spirit teach its own operating manual? Any uncertain answer is a reject-and-remand. Aggressive lean: when a record mixes a thin directive with such matter, treat the WHOLE thing as Matter — borderline goes out; the directive can be re-captured cleanly on its own later. Matter is durable-but-misplaced concrete content; ephemeral or task-state content is NonIntent instead (per the durable-intent clause above), never Matter. For a mutation, the same arrow must survive: a redirect / inversion / hardening -> ClarifyTramples; a silently dropped clause -> ClarifyLosesMeaning (including a multi-target Supersede collapsed into fewer replacements than the retired set's distinct arrows).
7. AFFIRMATIVE GUIDANCE. Intent records should state the positive shape to follow: what the practice is, what the component does, what spelling/name/contract is canonical, or what boundary holds. If the operative guidance is primarily an exclusion, prohibition, forbidden wording list, or definition-by-negation, reject NegativeGuideline and remand for affirmative rewording. A record may mention a rejected old wording or forbidden example only when the positive rule remains the center.
8. CLASSIFICATION. Domain is a universal subject (a named instance is a referent) -> UnclearDomain; mis-filed or unsafe privacy -> UnclearPrivacy.
9. METADATA BURDEN (the signature gate). Does direct testimony or argued evidence support the claimed metadata rungs? Direct psyche declaration supports the named Certainty, Importance, or Privacy rung. Without direct declaration, Certainty follows modal strength; elevated Importance requires recurrence, architectural centrality, blast radius, or blocking effect; Privacy must be safe for the content. Unsupported Certainty -> Overstated. Unsupported elevated Importance -> ImportanceUnsupported. Unsafe or unclear Privacy -> UnclearPrivacy.
10. CROSS-RECORD COLLISION (against the live bundle). The same forward arrow already lives -> Duplicate. The candidate negates a live psyche arrow with no authorizing quote -> Contradiction. The candidate negates a live psyche arrow with authorizing testimony but without a maintenance operation resolving the live conflict -> InsufficientWarrant, remand for Supersede, ChangeRecord, Clarify, Retire, or ResolveClarification as appropriate.
